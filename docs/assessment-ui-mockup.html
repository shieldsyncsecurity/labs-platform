Cloud Security Engineer — hands-on assessment

Harden the data pipeline

You've just joined the team that runs this AWS account. It powers a small data pipeline: two S3 buckets, a processor function, and a pipeline service user. A security review flagged the account as risky — your job is to harden it without breaking the pipeline. Work in any order. You don't need to finish everything for a strong result — quality beats coverage.

Your objectives · suggested time in brackets

1 · Stop the public data exposure — at the source

[~12 min]

One of the buckets is readable by anyone on the internet. Block public access at the account level and remove whatever makes the bucket public — don't just mask it. Keep the legitimate access rules intact.

2 · Enforce encryption & secure transport

[~10 min]

Make sure everything written to the data bucket is encrypted at rest, and that the bucket refuses plain-HTTP requests.

3 · Right-size the pipeline user's permissions

[~12 min]

The pipeline-svc user has far more access than it needs. Scope it down to exactly what the pipeline requires — it must still be able to do its job.

4 · Turn on the flight recorder

[~8 min]

There is no API audit logging in this account. Set up a trail that covers all regions, can prove its logs weren't tampered with, and doesn't leak its own log files.

5 · Get the secret out of the code path

[~12 min]

The processor function carries a database credential in the environment variable SECRET_VALUE. Move it to the proper secrets store and point the function at it — it already knows how to read SECRET_ARN. The pipeline must keep working afterwards.

Bonus · Baseline threat detection

[~2 min]

If you have time: switch on managed threat detection for the account.

🧪 Test the pipeline — anytime

The processor function must still work when you submit. Run this as often as you like (console or CloudShell):

aws lambda invoke --function-name sslab-processor out.json

A healthy run returns "status": "ok".

Ground rules

  • Docs & AI assistants are allowed. Use whatever you'd use on the job.
  • Work alone. Another person helping you is a disqualifier.
  • Your session activity in this account is recorded for grading and integrity review.
  • Grading looks at the final state of the account — there's no answer box to paste into.
  • Partial credit is real. An unfinished objective done carefully beats a rushed sweep.

When you're done

You'll answer two short written questions (5 min): what you fixed first and why, and one weakness you noticed that wasn't on the list. These don't change your score — the employer reads them as written.

Auto-submits when the timer ends. Your work so far is graded either way.
