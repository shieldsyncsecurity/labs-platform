## Wave 1 — engine (`engine/ent-handler.mjs` + `engine/entinfra.mjs`, separate ent Lambda)

E1. **Report-token lifecycle** (first-sale blocker)
   - `GET /ent/report`: 404 if `assessment.reportRevokedAt` set or `reportExpiresAt` < now.
   - `GET /ent/report/candidate`: 404 if `invite.candidateReportRevokedAt` set or expired.
   - New `POST /ent/report/revoke` `{ assessmentId }` or `{ inviteToken }` → stamps revokedAt.
   - New `POST /ent/report/renew` same targets → clears revokedAt, sets expiresAt = now + 90d.
   - New assessments/invites get `reportExpiresAt = createdAt + 90d` at creation. Existing rows
     (no field) = valid; revocation always honored.
E2. **Erasure cascade** (first-sale blocker): `eraseCandidatePii` also stamps
   `candidateReportRevokedAt` so `/r/c/[token]` 404s after a DPDP/GDPR erase.
E3. **Report completeness**: `/ent/report` response gains ALL non-revoked invites of the
   assessment as rows with real status (Invited / Scheduled / In progress / Submitted / Expired);
   `candidateReportToken` included only on submitted rows (employer already holds the org token).
E4. **Money loop**: `markOrderPaid` CAS created→paid now ALSO atomically ADDs `order.credits` to
   the org (exactly-once via the same CAS). `POST /ent/orders` unchanged (admin UI will call it).
E5. **Low-credit trigger**: after a successful invite charge, if usage crosses ≥80% and
   `org.lowCreditNotifiedAt` unset for this threshold-cross → SES email to info@ + stamp. Reset
   the stamp when credits are added (top-up).
E6. **Dispute path**: new `POST /ent/problems` `{ inviteToken, message ≤2000, actor }` → appends
   `{ ts, message, actor }` to `invite.problems[]` (cap 10, oldest dropped) + SES to info@.
E7. **Slot on resume**: sanitized `GET /ent/invite` response includes `slotKey` once booked.
E8. **Auto-submit flag**: `/ent/submit` accepts `auto: true` → stored `autoSubmitted` on result.
E9. **Audit identity**: `/ent/orgs/credits` + org create/delete/erase accept `actor` (staff email
   from the app session) and write it to the audit entries (replaces hardcoded "admin").
   `/ent/orgs/credits` also accepts optional `reason` ≤300 chars → audit.

Non-goals wave 1: MFA enforcement (owner-present change), agreement/DPA click-through (mechanism
wave 2, legal text needs owner review), payment gateway, multi-lab catalog (owner designs labs).
